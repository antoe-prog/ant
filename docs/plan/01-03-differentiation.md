# 1-3. 차별화 포인트 결정

**단계:** 1. 전략 수립 (A~C) — 목표·타깃·차별화

**산출물:**

- [경쟁 앱 비교 표 (3개)](../product/competitive-comparison.md)
- [전략적 차별점 (경험 차별 ≥2)](../product/strategic-differentiation.md) + 코드 `DIFFERENTIATION` (`services/gateway/app/product_vision.py`)

## 1-1·1-2와의 정렬

- [1-1 한 줄 비전](../product/product-vision-one-liner.md)이 말하는 **가치**와 차별 문장이 **같은 방향**인지 본다.  
- [타깃 세그먼트](../product/target-segments.md)의 **우선 세그먼트**가 차별 경험을 **실제로 체감**할 수 있는지 확인한다.  
- [반복 시나리오](../product/recurring-scenarios.md) **S1~S3** 중 최소 하나가 차별 경험을 **드러내게** 설계되었는지 본다.

## Step 요약

- 경쟁 앱 3개를 빠르게 분석해 기능·가격·UX를 표로 정리한다.
- 기능이 아닌 “경험 차별화(톤·속도·맥락 유지·오프라인 연동)”를 최소 2개 이상 정의한다.
- 위 산출물 표·본문을 갱신한 뒤 `DIFFERENTIATION`·API 응답과 **모순 없이** 맞춘다.

## 완료 체크

- [x] 비교 표에 **3개 앱**이 채워져 있고, 각각 **한 줄 이상** 근거가 있는가  
- [x] **경험 차별화**가 2개 이상이며, **기능 나열**만으로 끝나지 않았는가  
- [x] `strategic-differentiation.md` / `product_vision.DIFFERENTIATION` / `GET /v1/product-vision`이 **같은 주장**을 하는가  
- [x] 1-1 비전·1-2 우선 세그먼트와 **정면 모순**이 없는가
