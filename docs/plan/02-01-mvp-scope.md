# 2-1. 필수 기능 정의 (MVP 범위 확정)

**단계:** 2. 제품 스펙·아키텍처 설계 (D~F) — 요구사항·경험·구조

**산출물:**

- [Product Scope — V1 MVP·후순위](../product-scope.md#v1-mvp-scope) (필수 ≤5 · 후순위 · 횡단 관심사 한눈에)
- 세부 스펙: [AI 채팅](../product/feature-ai-chat.md) · [대화 히스토리](../product/feature-conversation-history.md) · [설정](../product/feature-settings.md) · [비기능 목표](../product/non-functional-targets.md)

## SSOT 역할 (Core vs V1 표)

- **문장 계약:** `product-scope.md` **한 줄 요약**의 **Core** 한 줄이 “V1에 무엇이 들어가는가”의 **요약 SSOT**다.  
- **표:** `## V1 MVP·후순위`의 **필수** 행은 Core와 **같은 범위**를 표로 고정한 것이다(숫자만 맞추는 중복이 아님).  
- **바꿀 때:** 범위를 줄이거나 늘리면 **Core를 먼저** 고치고, 같은 PR에서 V1 표·관련 `feature-*.md`를 맞춘다.

## “기능 1개” 정의

- **능력 묶음** 기준: 사용자가 이름 붙일 수 있는 **한 가지 제공 가치**를 1개로 센다.  
  - 예: 검색·공유·삭제는 모두 “대화 히스토리” **1개**로 묶는다.  
  - 예: 모델·언어·응답 톤은 “설정” **1개**로 묶을 수 있다.  
- **화면 개수**나 **API 엔드포인트 개수**로 세지 않는다.

## 1-x·스코프와의 정렬

- [한 줄 비전](../product/product-vision-one-liner.md)·[반복 시나리오 S1~S3](../product/recurring-scenarios.md): V1 필수만으로 **첫 가치**(시나리오 중 최소 1개)를 끝까지 재현할 수 있는가?  
- [우선 타깃](../product/target-segments.md): 필수 기능이 **우선 세그먼트**의 페인을 직접 줄이는가?  
- [차별·경쟁](../product/strategic-differentiation.md), [경쟁 비교](../product/competitive-comparison.md): V1에서 **과장 없이** 낼 수 있는 차별만 남겼는가?

## Step 요약

- 필수를 **5개 이내**로 적는다. 이 템플릿의 기본 계약은 `product-scope.md` **Core** 한 줄과 맞춘다.  
- “없어도 런칭 가능한 기능”을 **후순위**로 같은 섹션에 명시한다.  
- 성능·예산·프라이버시·관측 등 **횡단** 항목은 기능 5개와 **섞어 세지 않는다.** 정의·목표는 [비기능 목표](../product/non-functional-targets.md)·[5-2 관측](05-02-observability.md) 등에서 다룬다.  
- **예외(런칭 게이트):** 최소 관측(핵심 오류 로그·지연 메트릭)·장애 시 사용자 안내(빈 응답·재시도)는 “기능 5개”에 넣지 않되, **V1 출격 조건**으로는 포함할 수 있다 — 세부는 비기능·운영 문서에 한 줄이라도 적는다.

## 이 저장소용 예시 (소셜 앱 예시 대신)

- **필수에 넣기 좋은 조합:** AI 채팅(스트리밍/오류 처리) · 대화 히스토리 · 모델·언어·응답 스타일 설정.  
- **후순위로 빼기 쉬운 것:** 계정 연동(소셜 로그인), 푸시 알림, 실험 대시보드, 멀티 프로필 등.

## 2-2와의 연결

- [2-2 UX 플로우](02-02-ux-journey.md)의 5~7단계는 **여기서 정한 필수 기능**만 등장하게 맞춘다. 단계 표 산출물은 [first-value-journey.md](../product/first-value-journey.md)에 둔다.

## 완료 체크

- [ ] 필수가 **5개 이하**이고, 각 항목이 **능력 묶음** 정의에 맞는가  
- [ ] `product-scope.md`의 **필수·후순위**가 이 단계 결정과 **같은 PR에서** 갱신되었는가  
- [ ] S1~S3 중 **최소 1개**가 V1 필수만으로 완결되는가  
- [ ] 후순위에 밀린 일이 **런칭 후에도 안전한지**(법·보안·데이터) 한 번 점검했는가  
- [ ] **최소 관측·장애 시 UX**를 V1 게이트에 넣기로 했다면 비기능·운영 문서에 **반영**했는가
