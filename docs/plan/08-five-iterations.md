# 8. 최적 판단·5회 반복 검증 방식

**단계:** 실행 계획 전체를 다듬는 메타 절차

- **반복 1회차 (초안)**: 위 A~Z 단계를 모두 나열해 누락 영역을 찾는다. (예: 관측·알림 누락)
- **반복 2회차 (단순화)**: 각 단계에서 “지금 단계에서 꼭 필요한가?”를 기준으로 항목을 30% 줄인다.
- **반복 3회차 (리스크 중심)**: 장애·비용·법적 리스크 관점에서 항목을 다시 정렬하고, 상위 3개 리스크 완화 항목을 강화한다.
- **반복 4회차 (리소스 현실화)**: 팀 인원·예산·기간을 고려해 “이번 분기 내 현실적으로 가능한 범위”만 남긴다.
- **반복 5회차 (실행 체크리스트화)**: 남은 항목을 JIRA/이슈 트래커에 태스크로 쪼개고, 담당자·기한·완료 조건을 붙인다.

이 실행 계획은 개인 모바일 생성형 AI 앱을 A~Z 단계로 끌어가되, 5회의 반복 검증을 통해 “실제로 실행 가능한 최소 세트”로 지속적으로 다듬는 것을 목표로 한다.
