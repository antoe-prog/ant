# 모듈 초안 표 (2-3 산출물)

[2-3 기술 아키텍처 초안](../plan/02-03-tech-architecture-draft.md)에서 정한 **모듈 경계**를 여기 표로 고정한다. 다이어그램은 [system-diagram](system-diagram.md), 경계 원칙은 [boundaries](boundaries.md)와 **모순 없이** 맞춘다.

<a id="draft-module-matrix"></a>

## 이 템플릿 기본선

- **LLM 호출·프록시:** 별도 마이크로서비스로 쪼개지 않고, **[API 게이트웨이](boundaries.md) 한 덩어리** 안에서 처리한다(`services/gateway`). 팀이 프록시만 분리하면 **행을 추가**한다.  
- **벡터 DB / RAG 스토리지:** [3-x 데이터](../plan/03-03-personalization-data.md) 등에서 도입할 때만 채운다. **MVP에 없으면 아래 표에서 N/A**로 둔다.

## 모듈 × 배포 × 스케일링

| 모듈 | 책임(한 줄) | 이 레포·경로 | 독립 배포 | 스케일링(수평/수직) |
|------|-------------|----------------|-----------|---------------------|
| 모바일 클라이언트 | UI·로컬 상태 | `apps/ios`, `apps/android` | 스토어 단위(앱별) | 클라이언트—**해당 없음** 또는 기기 성능 |
| 운영 Admin 웹 | 브라우저 기반 백오피스·대시보드(읽기 우선) | `apps/admin-web` | 정적 호스트 또는 CDN 단위 | **해당 없음**(정적) 또는 CDN 캐시 |
| API 게이트웨이(+ LLM 경로) | 인증·정책·프롬프트·**LLM 호출**·요청 로깅·**Admin용 CORS** | `services/gateway` | 예 | **수평** 권장(스테이트리스) |
| 데이터 스토어 | 세션·유저 등 영속/캐시 | 다이어그램상 Postgres/Redis 등 — 실제는 팀 인프라 | 저장소별 | DB·캐시 정책에 따름 |
| 벡터 DB·추가 스토리지 | 임베딩·RAG | **없으면 N/A** | N/A 또는 별 서비스 | 도입 시 기술 선택 |
| 관측·모니터링 | 로그·메트릭·(트레이스) | [5-2 관측](../plan/05-02-observability.md), 운영 문서 — [2-1 V1 게이트](../product-scope.md#v1-mvp-scope)와 연계 | 보통 공유 스택 | 수집 파이프라인·샘플링 |

- **독립 배포:** “이 컴포넌트만 롤백·버전업 가능한가” 수준으로 **예/아니오/조건부**로 적어도 된다.  
- **스케일링:** MVP에서는 **한 줄 메모**로 충분하다. 수치 SLO는 [비기능 목표](../product/non-functional-targets.md)와 맞춘다.
