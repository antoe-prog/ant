# 프롬프트·시스템 (3-2 산출물)

[3-2 계획](../plan/03-02-prompt-system.md)에서 정한 **시스템 프롬프트 구조·전처리 위치**를 여기에 요약한다. **문자열 SSOT**는 코드에 두고, 이 파일은 **의도·축·분업**을 맞추는 데 쓴다.

<a id="prompt-system-spec"></a>

## 코드 SSOT (게이트웨이)

- **기본 시스템 프롬프트:** `services/gateway/app/product_vision.py`의 `DEFAULT_SYSTEM_PROMPT`  
- **프롬프트 레지스트리:** `services/gateway/app/services/prompt_registry.py` — `default`(위 상수)·`summary`·`translation` 등 키. 현재 채팅 경로는 주로 **`default`**를 사용한다. 키를 늘리거나 문자열을 바꿀 때는 **같은 PR**에서 이 md 요약을 맞춘다.

## 시스템 프롬프트 네 축 (역할·톤·금지·포맷)

| 축 | 질문 | 비고 |
|----|------|------|
| 역할 | 누구의 조수인가, 무엇을 **우선**하는가? | [한 줄 비전](product-vision-one-liner.md)·`ONE_LINER`와 모순 없게 |
| 톤 | 짧은지, 격식인지, **다음 행동**을 유도하는지? | S1~S3·차별 문구와 정렬 |
| 금지 | 안전·법·브랜드 **거절** 범위는? | `DIFFERENTIATION`·정책과 맞출 것 |
| 출력 포맷 | 목록·단계·길이 제한 등 **기본 형태**는? | 모바일 한 화면 가독성 |

문구를 고칠 때는 **위 코드 파일의 문자열**을 편집하고, 비전·차별 md와 **같은 주장**인지 PR에서 본다.

## 전처리(언어·길이·금칙어) — **어디서 할지**

| 관심사 | 권장(서버) | 클라이언트(보조) | MVP |
|--------|------------|-------------------|-----|
| **길이 제한** | 게이트웨이에서 거절/절단 + [5-1](../plan/05-01-api-llm-proxy.md) 로그와 일치 | 입력창 UX(자릿수 안내) | **서버 우선** |
| **언어 감지** | 필요 시 게이트웨이 또는 모델에 위임 | UI 기본 언어와 맞추기 | **후순위 N/A** 가능 |
| **금칙어** | 정책 확정 시 게이트웨이 | 오탐·업데이트 부담 큼 | **없으면 N/A** 명시 |

**이중 적용 금지:** 같은 검사를 **앱과 서버 둘 다**에 넣지 않는다(한쪽은 얇게).
