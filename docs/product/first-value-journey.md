# 첫 실행 ~ 첫 가치 여정 (2-2 산출물)

[2-2 UX 플로우](../plan/02-02-ux-journey.md)에서 정한 **5~7단계**를 여기 표로 고정한다. 등장하는 능력은 [2-1 MVP](../plan/02-01-mvp-scope.md)·[Product Scope V1](../product-scope.md#v1-mvp-scope)의 **필수**만 쓴다.

**여정 전제:** **첫 설치 후 첫 세션**이 기본이다. 재방문 사용자는 [2-2](../plan/02-02-ux-journey.md)에 따라 **짧은 여정**을 아래와 구분해 적는다.

**첫 가치:** [반복 시나리오](recurring-scenarios.md) **S1~S3** 중 팀이 고른 **하나**를 “끝까지” 재현하는 순간(예: 첫 응답 완료, 히스토리에서 재개 등).

**등장 기능 열:** [한 줄 요약의 Core](../product-scope.md)에 대응하는 레이블만—`AI 채팅` · `대화 히스토리` · `모델·언어·응답 스타일 설정` (동의어·괄호 생략 OK). Core에 없는 이름(온보딩 등)은 **적지 않는다.** OS 권한만 있는 단계는 **공란**.

|---|------------|---------------------------|------------------------|---------------|

| 2 | | | | |

| 4 | | | | |
| 5 | | | | |

- **6~7행**은 필요 없으면 표에서 삭제한다.  
- **재방문** 여정은 표 아래에 **소표**로 두거나, “1~N행 스킵” 메모를 남긴다.

## 이 템플릿용 예시 (채워 넣고 팀에 맞게 수정)

| # | 단계 | 사용자 행동(1개) | 등장 기능 | 시나리오 |
|---|------|------------------|-----------|----------|
| 1 | 첫 화면 | 안내·권한이 있으면 **허용 또는 스킵** | _(공란 — Core 밖 이름 금지)_ | S1 등 |
| 2 | 대화 시작 | 입력창에 질문 작성 후 **전송** | AI 채팅 | |
| 3 | 응답 확인 | 스트리밍이면 **끝까지 읽기** 또는 중단 | AI 채팅 | |
| 4 | (선택) 다시 묻기 | **새 입력 전송** | AI 채팅 | |
| 5 | (선택) 나중을 위해 | **대화 저장** 또는 히스토리로 이동 | 대화 히스토리 | |

**재방문 예시(짧은 여정):** 같은 첫 가치가 “히스토리에서 재개”라면 예를 들어 2~3행만 남기고, 등장 기능도 위 레이블만 사용한다.

## 장애·빈 응답 (분기)

첫 가치 경로에 **넣지 않아도** 되나, [2-1](../plan/02-01-mvp-scope.md) **V1 게이트**로 잡았다면 한 줄이라도 적는다.

- 예: 전송 후 오류 → **재시도** 탭 한 번 · 빈 응답 → **다시 시도** 또는 안내 문구.

[비기능 목표](non-functional-targets.md)·[5-2 관측](../plan/05-02-observability.md)과 모순 없게 맞춘다.
