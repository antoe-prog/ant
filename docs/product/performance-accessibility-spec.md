# 성능·접근성 (4-3 산출물)

[4-3 계획](../plan/04-03-performance-accessibility.md)의 **클라이언트 목표**와 [비기능](non-functional-targets.md)의 **게이트웨이 p95**를 구분해 적는다.

<a id="performance-accessibility-spec"></a>

## 지표 구분 (모순 방지)

| 구분 | 목표 예시 | 측정 위치 |
|------|-----------|-----------|
| **클라 UX** | 첫 화면 렌더 ~2초, 주요 탭 응답 ~1초 | 기기·프로파일러 |
| **API·LLM** | 짧은 프롬프트 p95 3초 미만 | 게이트웨이·공급자 | 비기능 SSOT |

두 층을 **섞어 한 줄 목표로 쓰지 않는다.**

## 접근성 최소

- 폰트 스케일·**색 대비**(본문/버튼)·VoiceOver/TalkBack **레이블**을 체크리스트로 적는다.  
- WCAG 전체를 MVP에 강제하지 않되, **회귀 시 재측정** 항목을 남긴다.
