# LLM·모델 전략 (3-1 산출물)

[3-1 계획](../plan/03-01-llm-strategy.md)에서 정한 **주력·백업·(선택) 특수** 모델과 선택 근거를 여기에 둔다. 구현 매핑은 게이트웨이 `ChatRequest.model`·`/v1/models` 등과 **같은 PR**에서 맞춘다.

<a id="llm-model-strategy"></a>

## 분류 (필수 3종이 아님)

- **주력:** 일반 대화·시나리오 S1~S3 기본 경로.  
- **백업:** 주력 실패·레이트 리밋·타임아웃 시 **폴백** (동일 공급자 다른 모델 또는 다른 공급자).  
- **특수 태스크:** 요약·번역 등 **별 모델이 이득일 때만** — MVP에서는 **N/A**로 두고 행을 삭제해도 된다.

## 모델 표

| 역할 | 모델 ID·공급자 | p95 목표(짧은 프롬프트) | 단가·토큰 정책(스냅샷) | 가용 지역 | 비고 |
|------|----------------|-------------------------|-------------------------|-----------|------|
| 주력 | | [비기능](non-functional-targets.md)와 일치 | _(가격표 기준일: YYYY-MM-DD)_ | | |
| 백업 | | 동일 또는 완화 | | | |
| 특수 | **N/A** 또는 채움 | | | | |

- **단가·지역**은 변경이 잦다. 표 옆 또는 푸터에 **근거 조사 일자**를 적는다.  
- **p95**는 [비기능 목표](non-functional-targets.md)의 짧은 프롬프트 기준과 **모순 없이** 맞춘다.

## 백업이 **다른 공급자**일 때

주력과 백업의 **공급자(벤더)**가 다르면 아래가 달라질 수 있다. **백업 행의 비고**에 한 줄이라도 적거나, 별도 이슈로 남긴다.

- **비밀·설정:** API 키·엔드포인트·프로젝트 ID 등 **별도 자격 증명**  
- **관측·로그:** [5-1](../plan/05-01-api-llm-proxy.md) 수준에서 **provider / model 식별**이 폴백 경로에서도 일관되는지  
- **데이터·지역:** 가용 리전·DPA·보관 정책이 주력과 **동일한지** — 다르면 [비기능](non-functional-targets.md)·프라이버시 흐름과 **모순 없이** 문서화

## MVP 최소선

- **주력 + 백업** 두 행만으로도 V1 전략 문서는 **완료**로 본다.  
- 특수 태스크는 **후순위**로 두고 [Product Scope 후순위](../product-scope.md#v1-mvp-scope)와 말이 맞는지 본다.
